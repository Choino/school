Longest Common Subsequence

Introduction

A Longest Common Subsequence (LCS) of two strings is exactly what the name implies! For instance, an LCS of the strings haedlelfog and abchxeylzloabc is the string hello. Note that the characters of the LCS do not need to be contiguous in either of the original strings. Also, there may be multiple LCS's.

It is simple to find an LCS of two strings recursively. Suppose that s1 and s2 are strings, and c1 and c1 are characters. Define LCS(s1+c1,s2+c2) to be:

  • LCS(s1,s2)+c1 if c1==c2
  • LCS(s1+c1,s2) if c1!=c2 and LCS(s1+c1,s2)>LCS(s1,s2+c2)
  • LCS(s1,s2+c2) if c1!=c2 and LCS(s1,s2+c2)>LCS(s1+c1,s2)
Of course, LCS(s1,empty)=LCS(empty,s2)=empty.

A straightforward recursive implementation of this definition would be unusable, but a dynamic programming implementation should run in quadratic time. This is because for input strings s1 and s2, we need to complete a table with length(s1)*length(s2) entries, where each entry requires us to examine three previously computed entries and some simple arithmetic.

Requirements

You will write a class LCS in the file LCS.java with the method: 
    public static String findLCS(String s1, String s2)
that will find a longest common subsequence of two strings. It should run in less than ten seconds for strings up to length 100. Your code must work with my test file. Also include a brief README.txt file.
