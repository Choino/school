Goal: To utilize JQuery (and Javascript) in conjunction with some PHP server scripts and a MySQL database to implement a simple 2-player game of Hangman.

 

In this assignment you will implement a 2-player Hangman game as a Web application using JQuery / Javascript / AJAX on the client side and some PHP scripts and MySQL on the server side.  Your program will proceed in the following way:

 

Setting up the Game:

-      The server will place players together by 2's on a first-come first-serve basis.  Thus if 5 players, A, B, C, D and E, all connect to play Hangman, Player A will play Player B, Player C will play Player D and Player E will wait until another player connects. 

-      Players will be associated with new games via state variables stored in the database.  Think about how you can store and update these values in order to make the game work.  For a simple example of how the games might be set up, see handouts setGame.php, players.txt, getGame.html and findGame.php.

-      Players in an active game will be maintained on the server via session and database variables.  These will be used to keep track of whose turn it is, the current word, etc.  Note that session variables are local to each user but database entries can be shared by both users.  Think about where in your game each will be appropriate.

-      Players connect by logging in (see Assignments 2 and 3 for login requirements – like Assignment 3 you must have a new user option for this assignment).

-      Once connected players are assigned a game and once both players in a game have connected the game may start.

 

Playing the Game

-      Once both players are set to play a game, they will take turns making and guessing words.  This all must be done using JQuery and AJAX.  A lot of the work for this will be done on the server, but the responses will always be XML data rather than new web pages. To see a simple demonstration of AJAX communication between 2 clients, see handouts setChat.php, Chat.php, getID.php, getMessage.php and processMessage.php.

-      Assume for a game the players are Player 1 and Player 2 (connected in that order).  In the first round Player 1 will create a word and Player 2 will guess.  They will switch for round two, with Player 2 creating the word and Player 1 guessing.  The process will continue indefinitely.  The different roles can be determined via client-side state variables and function calls, coordinated by the server. 

-      For consistency in terminology, call the player who has created the word the MAKER and call the player who is guessing the work the GUESSER.

-      Initially a prompt is shown to the MAKER, and the MAKER types in a word and submits it (via AJAX).  This word will then be stored on the server (in the DB) and a message will be sent to the GUESSER (or, more correctly, a message will be made available for the GUESSER to retrieve) that he may start guessing.

-      Once the word has been submitted, for the remainder of the round the MAKER will be passive and the GUESSER will be active.  However, the MAKER should still poll the server for messages in order to keep the status of the game consistent between the two users.  In order to do this, for each response the server makes to the GUESSER, the server should store the same data in the database for the MAKER to download.  See the Chat.php handouts (total list shown above) for an idea of how to do this.

-      After the word is submitted, both clients should show the round as it progresses.  Specifically, they should show:

-      The current word, with any letters that have been guessed filled in and with blanks (or some other clear designators) for the letters that have not yet been guessed.

-      The number of incorrect guesses that remain

-      The list of letters that have been already guessed

All of this information should be nicely formatted using CSS.

-      The GUESSER will have 7 incorrect guesses to guess the word (the GUESSER is allowed an indefinite number of correct guesses).  With each guess (submitted via AJAX) the server will check the guessed letter against the letters in the word and return the following information in an XML document (you must define your own XML language for this)

-      Whether or not the letter was found in the word

-      The "current" word with any correctly guessed letters filled in (if a letter is in the word more than one time all occurrences should be filled in)

-      The number of incorrect guesses remaining

This information should also be made available to the MAKER via one or more entries in the database.  The next poll from the MAKER should receive this data as a response.  When either player receives this information, his local display of the game should be updated accordingly.

-      The GUESSER will locally keep track of the letters that have been guessed so far, and this list will be checked before submission of a new guess.  Any repeat guesses of letters should not be submitted to the server.

-      If 7 incorrect guesses are made the GUESSER loses that round.  An XML message should be sent to indicate this and both the GUESSER and MAKER should display this result.

-      If all letters are filled into the word the GUESSER wins that round.  An XML message should be sent to indicate this and both the GUESSER and MAKER should display the result.

-      However the round ends, the GUESSER and MAKER should then switch roles and proceed with the next round. 

-      You do not have to handle a graceful termination of the game (i.e. the game never logs out the players or deletes their game DB information).  However, you can do this for extra credit.

Implementation Details:

-      You must implement the game as described above using JQuery (and Javascript) on the client and using several PHP scripts and a MySQL database on the server.

-      Both clients should run the same programs – the difference in the players should be determined by state variables.

-      After logging in, all interaction between the clients and the server must be done via AJAX requests (no hard page refreshes are allowed).

-      The word to be guessed is generated by the MAKER client and stored on the server, but it should NOT be sent to the GUESSER client.  Note that this is different from the LINGO game and this difference prevents the client from cheating.

-      For help with JQuery, see the course notes and handouts, and the many Web references on the subject.

-      To help the TA with the grading, you must write an init.php script that sets up the game for playing (including reinitializing the database).  This will wipe out all game data for any active games and reset the DB for new games to be played.

-      To further help the TA with grading, include your player ids and passwords in your index.html file.
